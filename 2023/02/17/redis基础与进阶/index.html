Nosql和SQL的介绍

概述：数据库存储类型有分为SQL（关系型）和NOSQL（非关系型）型数据库

SQL型的数据库存储特点：

（ 1 ）结构化：表结构存储

（ 2 ）表之间是有关联的

（ 3 ）可以使用SQL语句进行操作

（ 4 ）满足ACID特性：原子性、永久性、一致性、隔离性

（ 5 ）存储方式：磁盘

NOSQL型数据库存储特点：

（ 1 ）非结构型：像redis数据库是以key-value结构进行存储的

（ 2 ）无关联的

（ 3 ）非SQL

（ 4 ）存储方式：内存（性能比较好）

使用场景：

（ 1 ）SQL型：对数据安全性和一致性要求较高，数据结构固定

（ 2 ）NOSQL型：数据结构不固定，对性能要求较高，对一致性和安全性要求不高

Redis的基本介绍

概述：Redis诞生于2009年全称是Remote Dictionary Server，远程词典服务器，是一个基于内存的键值型NoSQL数据库。

Redis的特征：

（ 1 ）键值性（key-value）：value支持多种数据结构，功能丰富

（ 2 ）单线程，每个命令具备原子性

（ 3 ）低延迟，速度快（基于内存、IO多路复用、良好的编码）

（ 4 ）支持数据持久化

（ 5 ）支持主从集群、分片集群支持多语言客户端

String类型的相关命令

String类型，也就是字符串类型，是Redis中最简单的存储类型。其value是字符串，不过根据字符串的格式不同，又可以分为3类

  • string：普通字符串

  • int：整数类型，可以做自增、自减操作

  • float：浮点类型，可以做自增、自减操作

不管是哪种格式，底层都是字节数组形式存储，只不过是编码方式不同。字符串类型的最大空间不能超过512m。

命令 作用 使用形式
set 添加或者修改一个String类型的键值对 set key value
get 获取已存在key对应的value get key
mset 批量添加或修改String类型的键值对 mset k1 v1 k2 v2 …
mget 批量获取多个key对应的value mget k1 k2 k3 …
incr 让一个整型的key自增1 incr key
incrby 让一个整型的key自增并指定步长 incrby key 步长
incrbyfloat 让一个浮点类型的数字自增并指定步长 incrbyfloat key 步长
setnx 添加一个String类型的键值对，前提是这个key不存在，否则不执行 setnx key value
setex 添加一个String类型的键值对，并且指定有效期 setex key value

（ 1 ）Redis的key允许有多个单词形成层级结构，多个单词之间用∵’隔开，格式【项目名:业务名:id】，可以根据实际需要进行改变

（ 2 ）如果Value是一个ava对象，例如一个User对象，则可以将对象序列化为JSON字符串后存储

List类型相关的命令

Redis中的List类型与Java中的LinkedList类似，可以看做是一个双向链表结构。既可以支持正向检索和也可以支持反向检索。

特征也与LinkedList类似：有序、元素可以重复插入和删除快查询速度一般。

命令 作用 使用形式
lpush 向左列表插入一个或多个元素 lpush key element …
lpop 移除并返回列表左侧的第一个元素，没有则返回nil lpop key
rpush 向右列表插入一个或多个元素 rpush key element …
rpop 移除并返回列表右侧的第一个元素，没有则返回nil rpop key
lrange 返回一段角标范围内的所有元素 lrange key start end
blpop 在没有元素时等待指定时间，而不是直接返回nil blpop key
brpop 在没有元素时等待指定时间，而不是直接返回nil brpop key

Set类型的相关的命令

Redis的Set结构与Java中的HashSet类似，可以看做是一个value为null的HashMap。因为也是一个hash表，因此具备与HashSet类似的特征：

  • 无序
  • 元素不可重复
  • 查询块
  • 支持交集、并集和差集操作

常见的命令如下

scard 返回set集合中的成员个数 scard key
sismember 判断某个成员是否在集合中 sismember key member
smembers 获取set集合中的所有成员 smembers key
sinter 求key1和key2的交集 sinter key1 key2
sdiff 求key1和key2的差集 sdiff key1 key2
sunion 求key1和key2的并集 sunion key1 key2

SortSet类型相关的命令

Redis的SortedSet是一个可排序的set集合，与Java中的TreeSet有些类似，但底层数据结构却差别很大。SortedSet中的每一个元素都带有一个score属性，可以基于score属性对元素排序，底层的实现是一个跳表（SkipList）加hash表。SortedSet具备下列特性：

  • 可排序
  • 元素不重复
  • 查询速度快

因为SortedSet的可排序特性，经常被用来实现排行榜这样的功能。

命令 作用 使用形式
zadd 添加一个或多个元素到sortedset，如果已经存在则更新其score值 zadd key score member …
zrem 删除指定的元素 zrem key member …
zscore 获取sorted set中的指定元素的score值 zscore key member
zrank 获取sorted set 中的指定元素的排名 zrank key member
zcard 获取sorted set中的元素个数 zcard key
zcount 统计score值在给定范围内的所有元素的个数 zcount key mim max
zincreby 让sorted set中的指定元素自增，步长为指定的increment值 zincreby key increment member
zrange 按照score排序后，获取指定排名范围内的元素 zrange key mim max
zrangebyscore 按照score排序后，获取指定排名范围内的元素（按照分数排名） zrangebyscore key mim max

注意：所有的排名默认都是升序，如果要降序则在命令的Z后面添加REV即可
